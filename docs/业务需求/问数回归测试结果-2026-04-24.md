## 9. 总结结论

### 9.1 覆盖性结论

当前 TiDB workspace 下：

- instruction 已导入：**14**
- sql_pair 已导入：**0**

因此当前系统对 `docs/业务需求` 的覆盖状态是：

- **规则覆盖：已导入，但运行时命中效果很差**
- **SQL 模板覆盖：基本未形成可用 sql_pair 主链路**

### 9.2 准确性结论

当前系统**不能判定为“已覆盖并且可以准确问数”**。

本轮综合判断：

1. **UI 主链路阻塞**：thread detail / response 接口 404，Home UI 无法作为稳定验证链路。
2. **规则命中不足**：13 条可直接提问规则里仅 1 条（R05）稳定命中 instruction。
3. **模板准确率低**：11 个 draft_sql 模板中仅 3 个通过。
4. **缺失外部数据源兜底失败**：T05/T14/T15 都出现错误替代或臆测。
5. **图表可生成，但不等于结果准确**。
