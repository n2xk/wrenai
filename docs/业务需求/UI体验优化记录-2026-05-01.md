## 结论摘要

本轮没有发现导致 UI 全站不可用的 P0 问题，但问数结果状态同步、SQL 方言/预览链路、诊断默认列表、反馈列表信息密度等仍存在明显体验与可观测性问题。以下问题需要进入后续修复队列。

## 问题清单

| 严重度 | 场景 | 现象 | 影响 | 建议 | 证据 |

| P1 | 问数结果生成 | 页面发起“综合日报”问数后，后端已完成并生成结果，但前端一段时间内仍显示“进行中 / 正在生成 SQL”；刷新页面后才显示最终预览与结果。 | 用户会误以为任务卡死，回归用例也难以稳定判断完成态。 | 检查 thread response polling / streaming 结束事件；完成后主动刷新 response detail 与 workbench artifact。 | Playwright：`/home/1?...`；response id=1；刷新后恢复。 |

| P1 | API scope / 数据看板 preview | `POST /api/v1/dashboard-items/1/preview` 带完整 KB scope 时返回 `Dashboard item not found`，只带 `workspaceId` 时通过。 | 数据看板是 workspace 级资产，但部分调用容易继承问数/知识库 scope，造成切换页面后误报 not found。 | 统一 Dashboard API scope 规则：列表、详情、preview 都按 workspace scope；前端路由切换时不要携带 KB-only 约束。 | 产物化 smoke 首次 404，修正脚本为 workspace scope 后通过。 |

## 本轮已验证的正向体验

- 反馈管理页可以展示跨 workspace 可见反馈，并能看到正反馈与负反馈。
- 调用诊断在 `apiType=ASK` 筛选下可以看到 thread ID、问题文本、`nl2sql` 路径与详情 JSON。
- 数据表保存后可以预览；数据表 AI F-01/F-04/G-01 可保存新版本并记录 History，失败分支不会写入新版本。
- 数据看板 item 与数据表 item 的重命名/删除链路已补测；表格结果固定到看板会被明确拦截。
- 产物化只读 smoke 修正后，数据表列表/预览、看板列表/preview、反馈 API 均可访问。
