# UI 体验优化记录 - 2026-05-03

## 本轮观察范围

本轮观察来自 TiDB 干净全量回归 B0~B6，执行方式为 UI E2E / 同源 UI API 链路，证据目录见 `docs/业务需求/问数回归测试结果-2026-05-03.md`。

## 已验证通过的体验链路

1. 11 张 DEGRADED 数据表均可从问数结果保存到「数据表」，并可进入详情、查看 SQL、导出 CSV / Excel。
2. 图表生成完成后可固定到数据看板；本轮重试后 `dashboardItemId=1`。
3. 正/负反馈接口可用；负反馈 reason 应使用当前合法枚举 `incorrect_data_retrieved`。
4. 数据表 AI 操作 FILTER / CLEANING / GROUPING / ENRICHMENT 在 B3 重跑中通过。
5. 需要补充租户、渠道、时间、周期等条件的问题会先阻断，不会直接生成不可靠 SQL；补齐条件后的业务问数可继续生成 SQL。

## 本轮已处理

1. 图表 schema 仍在生成中时，前端禁用固定到看板，并补充后端 409 兜底，避免未完成图表被固定。
2. B2/B5 自动化关键字断言增加同义词归一要求，覆盖“首充/首存”“二存/二次存款/第2次充值”“非TOP3/NON_TOP3”等表达差异。
3. 补槽链路补充多轮前值携带断言：后续补充周期等新槽位时，已补充的租户、渠道等 resolved slots 会继续随请求提交，并展示在问题摘要中。

## 需要优化的问题

| 优先级 | 问题 | 影响 | 建议 |
| --- | --- | --- | --- |
| P1 | FULL 外部指标没有结构化补值入口 | PV/UV/下载点击 UV/投放金额相关 FULL 只能阻断 | 增加外部指标表格录入/上传或外部依赖数据集接入后，再做 FULL 同形回归 |
| P3 | console warning：antd Modal `destroyOnClose` deprecated | 非阻塞，影响控制台噪音 | 后续统一替换为 `destroyOnHidden` |

## 结论

本轮没有发现阻塞 B1 数据表资产化、B2/B4/B5 补齐条件后问数、B3 产物化能力的 UI P0 问题。FULL 同形覆盖的主要阻塞不在 UI 展示层，而是外部指标数据输入/补值链路尚未产品化。
