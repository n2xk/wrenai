## 5. 本轮补修问题

### F_T04 追问低 margin 时错误降级为 trusted reference

现象：

- 父问题 `T04` 可正确命中 `anchored_template` 并生成 `2026-04-01 ~ 2026-04-03` 的 D1-D7 cohort 累计收入。
- 追问「那只看 2026-04-02 的首存 cohort 呢？」在 top template 与竞争模板低 margin 时，错误降级为 `trusted_reference`，导致 LLM 生成 SQL 并触发 Trino parse error。

根因：

- 父轮生成 SQL 经过 table grounding 后，CTE / 表名带上 `tidb_business_demo_` 前缀。
- 模板样本中的 CTE / 表名没有该前缀。
- 历史 SQL 与当前模板的上下文连续性判断只做精确交集，未把 `seq` 与 `tidb_business_demo_seq`、`dwd_order_deposit` 与 `tidb_business_demo_dwd_order_deposit` 视为同一逻辑关系名。

修复：

- 在 `build_template_decision()` 中保留“参数已从追问 + history 补齐，且历史 SQL 与 top template 上下文连续”的 anchored path。
- 增强 `_matches_history_template_context()` 的 relation/CTE 名称归一化：当历史 SQL 使用一致物理前缀时，可按 suffix 映射回逻辑模板名再比较。
- 增加真实前缀场景单测，覆盖低 margin follow-up 不降级。

涉及文件：

- `wren-ai-service/src/core/fixed_order_ask_runtime.py`
- `wren-ai-service/tests/pytest/core/test_fixed_order_ask_runtime.py`
