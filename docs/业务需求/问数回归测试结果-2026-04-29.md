## 9. 本轮发现并修复的问题

### 9.1 KB scoped dashboard item 预览/重命名/删除 404

现象：固定图表到看板后，`/api/v1/dashboard-items/{id}/preview`、PATCH、DELETE 在 KB scoped 请求下返回 404：

- `Dashboard item not found. id: 9`
- `Dashboard not found.`

原因：dashboard item 相关 API 校验时强制使用 workspace scope 查 dashboard，但本轮图表固定在 KB scoped dashboard 下。

修复：

- `wren-ui/src/server/api/dashboardRestShared.ts`
  - 导出并复用 `ensureDashboardForRequestedScope`
  - `ensureDashboardItemForScope` 改为按当前请求 scope 校验
- `wren-ui/src/pages/api/v1/dashboard-items/[id]/preview.ts`
  - 预览接口改为按 requested scope 获取 dashboard

复测：

- 新建 dashboard item=11
- preview rows=3
- PATCH displayName 成功
- DELETE 成功

验证：

- `cd wren-ui && yarn check-types`：通过

## 10. 仍需修复/收敛的风险

1. **OQ06/OQ08 普通问数路由错误**
   - OQ06 应按渠道聚合成功充值订单笔数/金额，却强锚 T01 渠道日报。
   - OQ08 应查询登录但无成功充值玩家，却强锚 T03 首存 cohort。
   - 建议：增强模板 negative cues / 普通 SQL 优先条件；对“直接基于订单表/登录但没有充值”等问题降低业务模板锚定权重。

2. **LING01 原始 smoke 缺少租户时未澄清**
   - 缺少 `tenant_plat_id` 时应走澄清或默认租户策略，而不是进入 `anchored_generated` 并生成不可解析 SQL。
   - 建议：missing parameter 必须先澄清；禁止缺必填参数时尝试拼接 anchored_generated SQL。

3. **OpenRouter/DeepInfra 429 外部波动**
   - T10 图表、OQ03、OQ07 首次均遇到 provider 429，重试后通过。
   - 建议：继续完善 Novita fallback / 用户可见重试提示 / 自动重试退避。

4. **T11 图表 `chartType=null`**
   - 图表状态为 `FINISHED`，但 chartType 为空，不利于回归判断和固定到看板类型选择。
   - 建议：chart detail 持久化时补齐 chartType 或提供 recommended display/type 映射。

5. **本轮不是全量干净回归**
   - 未重置系统数据、未重建 TiDB seed、未新建 workspace。
   - 最终不能声明“全量干净回归通过”，只能声明“复用 TiDB workspace 的非全量回归结果”。
