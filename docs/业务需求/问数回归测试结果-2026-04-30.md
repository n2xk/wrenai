## 本轮发现并修复的问题

### OQ05 生成 SQL 中的 MySQL DATE_ADD 预览解析失败

现象：

```sql
DATE_ADD('2026-04-07', INTERVAL 1 DAY)
```

在当前 Wren UI internal preview 的 Trino-style parser 中报：

```text
mismatched input '1'. Expecting: ')', ',', 'ORDER'
```

修复：

- 在 AI-service SQL post processor 中增加 deterministic retry。
- 当 preview 返回 `INTERVAL` 解析错误时，将常见 MySQL/TiDB day interval 函数转换为 parser 可接受的形式再重试：

```sql
DATE_ADD('2026-04-07', INTERVAL 1 DAY)
-- retry as
DATE_ADD('day', 1, DATE '2026-04-07')
```

验证：

- OQ05 单独回归通过。
- OQ01 ~ OQ10 全量 route 回归通过。
