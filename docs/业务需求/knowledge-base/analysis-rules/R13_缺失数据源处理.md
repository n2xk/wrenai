# R13 缺失数据源处理

## 规则内容

当问题涉及投放金额、访问PV、访问UV、下载点击UV，而当前知识库中无对应 SQL 模型或表时，不能直接编造，也不能用其他字段替代；应先在对话中向用户索取这些外部指标，再结合系统可查询的内部 SQL 指标输出结果。

建议执行顺序：

1. 先识别问题是否依赖以下外部指标：
   - 投放金额
   - 访问PV
   - 访问UV
   - 下载点击UV
2. 若依赖外部指标，则先向用户索取缺失数据，优先索取与当前问题同粒度的数据：
   - 日趋势类问题：按 `biz_date + channel_id` 提供
   - 区间汇总类问题：按查询区间汇总提供
   - ROI/cohort 类问题：按 cohort 对应统计周期提供投放金额
3. 用户提供后，再进行后续计算与展示：
   - 可查的内部指标继续走 SQL / sql_pair
   - 外部指标使用用户提供值补齐
   - 最终再输出数据表和图表
4. 若用户未提供，则要明确说明哪些字段暂时无法计算，但可先输出不依赖这些外部指标的内部结果。

对话中索取用户数据时，应明确说明所需字段、粒度与格式，例如：

- 综合日报 / 趋势图：请提供每个日期、每个渠道的 `投放金额 / PV / UV / 下载点击UV`
- ROI：请提供对应统计周期的投放金额
- 首存成本：请提供对应日期或区间的投放金额

补齐后的派生指标按以下公式计算：

- `首存成本 = 投放金额 / 首存人数`
- `UV下载率 = 下载点击UV / 访问UV`
- `UV注册率 = 注册人数 / 访问UV`
- `ROI = 渠道累计收入 / 投放金额`
