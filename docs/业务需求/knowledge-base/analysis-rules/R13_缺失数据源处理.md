# R13 缺失数据源处理

## 规则内容

问题依赖投放金额、访问PV、访问UV、下载点击UV，但知识库无对应 SQL 模型或表时，不能编造，也不能用内部字段替代。

处理规则：

1. ROI、首存成本、UV下载率、UV注册率等需要外部指标时，先向用户索取同粒度数据：日趋势按 `biz_date + channel_id`，区间汇总按查询区间，ROI/cohort 按 cohort 周期。
2. 用户提供外部数据后，再结合内部 SQL 指标计算并输出表格/图表。公式：`首存成本=投放金额/首存人数`、`UV下载率=下载点击UV/访问UV`、`UV注册率=注册人数/访问UV`、`ROI=渠道累计收入/投放金额`。
3. 用户未提供时，应说明暂无法计算哪些字段，但可先输出不依赖外部指标的内部结果。
4. 用户明确说“暂时不用外部数据 / 去掉投放金额、PV、UV、下载点击UV / 只展示系统内可查询原始指标”时，进入降级内部指标路径：不触发外部数据追问，不输出被去掉的外部字段和派生率，并说明这是降级结果，不等同于 FULL 覆盖。

## 作用报表

- 综合日报表
- ROI回收表

## 备注

当前确认缺失：投放金额、PV/UV/下载点击UV。目标不是拒答，而是在缺源时追问补数；用户选择降级时跳过阻断，只输出内部可查字段。
