# 第一期 Excel 示例表格同形覆盖测试结果（2026-05-01）

## 1. 测试结论

本轮测试目标是验证 `docs/业务需求/第一期数据报表需求V1.xlsx` 原始需求中示例表格是否已经被问数核心用例完整覆盖、是否能查出与 Excel 示例表格同形的结果，并在同批用例中保存为数据表资产。

- Excel 工作簿共有 **7 个 sheet**。
- `表格汇总` 是索引页，不作为问数结果验收表。
- 其余 6 个报表需求 sheet 中共有 **11 张**需要复现/保存的示例表格：FT01~FT11。
- 本轮通过 MCP Playwright 在当前 UI 会话执行/复核 FT01~FT11，结论：**未通过，不能声明“第一期 Excel 示例表格已经完全覆盖且查出来一样”。**

## 5. 主要缺口

1. **外部数据补齐链路没有真正参与 SQL 生成**：FT01/FT02/FT04 即使在问题中内联投放金额、PV、UV、下载点击 UV，仍被策略判定为缺外部指标并阻断；FT03/FT05 是累计收入表，本身不应强依赖投放金额，也被误阻断。
2. **同形宽表 / 透视表能力不足**：FT09~FT11 能生成 SQL，但输出仍是长表或英文技术列，不是 Excel 要求的中文列序、宽表、透视列、汇总/分层行。
3. **复杂同形表生成会卡住**：FT06/FT07/FT08 进入 TEXT_TO_SQL 后长期停留在 PLANNING/GENERATING/CORRECTING，本轮超过工具等待窗口后手动停止，未产生 SQL。
4. **数据表保存不是核心瓶颈**：只要有 SQL 且有返回行，保存接口可工作；但保存的是非同形结果，不能算通过。

## 6. 后续修复建议

1. 为 FT01~FT11 建立明确的“报表模板 / 展示层变换”能力，不能只依赖通用 LLM SQL 生成；尤其要支持宽表、透视表、汇总行、环比行和固定枚举列。
2. 修正问数策略对外部指标的判断：允许用户在自然语言中补充外部指标后进入 SQL/结果组合；累计收入表不要误判为需要投放金额。
3. 对 FT06~FT08 的长 SQL 生成增加超时诊断、失败落库和用户可见错误，避免 UI 一直卡在过程态。
4. FT09~FT11 的保存资产目前只能作为“失败证据资产”，不能作为验收通过资产；修复同形输出后需要重新保存并覆盖验收记录。

## 7. 验收口径

任何一张 FT 表未同形输出或未成功保存，都不能声明 `第一期数据报表需求V1.xlsx` 示例表格全覆盖。本轮结果为 **0/11 同形通过，3/11 保存了非同形结果，整体未通过**。
