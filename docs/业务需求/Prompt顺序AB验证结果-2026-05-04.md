# Prompt 顺序 A/B 验证结果（2026-05-04）

## 验证目的

验证是否应将 Text-to-SQL prompt 默认顺序从：

1. `SQL SAMPLES`
2. `USER INSTRUCTIONS`

调整为：

1. `USER INSTRUCTIONS`
2. `SQL SAMPLES`

本轮只做代表性小样本验证，避免 OpenRouter 429；不替代全量 UI E2E 回归。

## 结论

本轮代表性 A/B 没有发现 Instruction-first 的显著收益：

- 通过率相同：A、B 均为 12/12 PASS。
- 模板决策与语义路由基本一致，没有证明 Instruction-first 能减少误锚定或误澄清。
- 耗时基本持平：平均耗时仅差 0.4s；中位数和 P90 反而 B 组略差。
- 个别 normal / trusted_reference 路径 SQL 文本不同，但都通过；没有构成切换默认顺序的证据。

因此当前建议：**保持默认顺序不改**，继续保留 `WREN_PROMPT_INSTRUCTION_FIRST_ENABLED` 作为实验开关。

## 后续建议

如果后续要进一步评估 Prompt 顺序，可扩大到：

1. 全量 B2~B6 UI E2E 回归对比。
2. 专门选择“业务规则强约束但 SQL sample 相似度高”的冲突样本。
3. 收集每条 case 的最终 SQL 差异、执行结果差异和 diagnostics 差异，按业务正确性而不是 pass/fail 做人工复核。

在没有新增证据前，不建议把 Instruction-first 设为默认。
