这份 runbook 用来配合下面几份文件做回归：

- 造数脚本：`docs/业务需求/seed.sql`
- 预期结果：`docs/业务需求/expected-results.md`

本轮目标分两层：

## 6. 常见排错清单

如果跑出来和预期不一致，按下面顺序排：

### 6.1 先看过滤条件

- 是否漏了 `tenant_plat_id = 990001`
- 是否漏了 `channel_id = 990011`
- 日期上界是否写成了 `<= end_date 23:59:59` 之外的别的口径

### 6.2 再看状态条件

- 充值是否只取 `status = 2`
- 提现是否只取 `status = 3`
- 投注是否只取 `settle_status = 1`

### 6.3 再看口径方向

- `charge_withdraw_diff = 充值 - 提现`
- `kill_rate = 输赢 / 有效投注`
- `bet_deposit_ratio = 有效投注 / 充值`
- `T04 渠道收入 = 输赢 - 洗码 - 任务 - 营销 - 优惠加扣款`

### 6.4 再看是否误把补零/NULL 改写了

- T10/T04 存在由递归 CTE 生成的补零行
- 某些比例在分母为 0 时应为 `NULL`
