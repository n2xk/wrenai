## 需要后续修复 / 优化

1. **FT02-D、FT08-D 的模板锚定偏差**
   - 现象：虽然问法明确“首存后第 N 日 / 相对天数”，最终 SQL 仍命中综合日报类宽表。
   - 建议：在问数策略或 SQL 模板 `businessSignature` 中强化 cohort/relative_day 意图，给综合日报模板增加 negative cues（如“首存后第1到第7日”“相对天数”“cohort累计”）。

2. **FT10-D 缺游戏类型维度**
   - 现象：推荐问法要求按游戏类型输出，但结果列没有 `game_type` / 游戏类型字段。
   - 建议：补充游戏类型流水分布模板或问数策略，要求包含游戏类型维度；同时给 FT09 类杀率投充比模板增加 negative cues（如“按游戏类型”“游戏类型流水分布”）。

3. **DEGRADED 用例 CSV 推荐问法需同步收敛**
   - 本轮补跑证明部分原始推荐问法容易触发外部依赖或澄清。
   - 建议将 FT01-D、FT02-D、FT03-D、FT04-D、FT05-D、FT07-D、FT08-D 的推荐问法改为本轮补跑口径：显式给出租户、渠道、日期、只用内部指标、避免出现“投放/ROI/PV/UV”等外部触发词。

4. **FULL 与 DEGRADED 应分开判定**
   - DEGRADED 已验证“可生成 / 可保存”的最小闭环。
   - FULL 仍需要验证是否与 Excel 示例表格同形、同口径，且 FT02/FT04 等依赖外部投放数据的项需要外部数据接入或明确替代口径。
