# 第一期数据报表需求全覆盖补充用例

## 1. 文档目的

本文件用于补齐 `docs/业务需求/第一期数据报表需求V1.xlsx` 相对当前 T01~T15 回归集的覆盖缺口。

当前 `docs/业务需求/问数回归测试计划.md` 已经覆盖核心问数主链路：

- 11 个可执行 SQL 模板：T01 / T02 / T03 / T04 / T06 / T08 / T09 / T10 / T11 / T12 / T13
- 3 个缺失外部数据阻断模板：T05 / T14 / T15
- 追问、图表、业务词典、外部依赖阻断、diagnostics

但要声明“覆盖 `第一期数据报表需求V1.xlsx` 的所有场景”，还需要补齐本文件定义的 **FC01~FC12** 增量用例。

> 口径说明：这里的“全覆盖”分两层：
>
> 1. **FC01~FC12**：覆盖业务问数场景、字段口径、分组维度、异常边界和外部数据补齐链路。
> 2. **FT01~FT11**：覆盖 `第一期数据报表需求V1.xlsx` 中 11 张示例表格的同形输出与数据表保存。详见 `docs/业务需求/第一期Excel示例表格全覆盖清单-2026-05-01.md` 和 `docs/业务需求/csv/15_第一期Excel示例表格全覆盖保存清单.csv`。
>
> 若最终报告要写“原始 Excel 示例表格已经完全覆盖且查出来一样”，不能只跑 T/FC 用例，必须额外执行 FT01~FT11。这里的“一样”指业务粒度、字段集合、列序、宽/长表方向、汇总行、分层行、周期列、公式口径一致；不要求 1:1 还原 Excel 的单元格样式、合并单元格、颜色和字体。

## 3. 全覆盖增量用例清单

### FC01：综合日报完整外部指标并表

| 项 | 内容 |
| --- | --- |
| 覆盖 Excel | 综合日报表 |
| 补齐缺口 | 投放金额、PV、UV、下载点击UV、UV下载率、UV注册率、首存成本 |
| 前置条件 | 用户在对话中或外部数据依赖入口补充 `biz_date + channel_id` 粒度的投放金额和流量指标 |
| 推荐问题 | 统计租户平台990001下渠道990011在2026-04-01到2026-04-06的渠道日报，并补充投放金额、PV、UV、下载点击UV、UV下载率、UV注册率和首存成本 |
| 外部输入样例 | `日期,渠道ID,投放金额,访问PV,访问UV,下载点击UV`；每行按日期+渠道提供 |
| 预期结果 | 在 T01 内部指标基础上并入外部字段；`UV下载率 = 下载点击UV / 访问UV`；`UV注册率 = 注册人数 / 访问UV`；`首存成本 = 投放金额 / 首存人数` |
| 通过标准 | 不再阻断；结果同时包含内部 SQL 指标和用户补充外部指标；公式列不编造、不错用其他字段 |

## 6. 全覆盖验收口径

只有同时满足以下条件，才能在测试报告中声明：

```text
第一期数据报表需求V1.xlsx 已完成全场景覆盖。
```

1. T01~T15 当前主链路全部通过。
