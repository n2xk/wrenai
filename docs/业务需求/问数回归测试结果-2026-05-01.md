## 4. 缺陷 / 风险清单

| 严重度 | 问题 | 影响 | 建议 |
| --- | --- | --- | --- |
| P1 | 普通 SQL 生成 TiDB 方言不稳定，出现 `TIMESTAMP WITH TIME ZONE`、反引号/Trino parser 不兼容等问题。 | PX01/PX05/LING01 人工核验失败；保存 SQL 模板、预览、图表、数据表 AI 操作都可能受影响。 | 统一 TiDB/MySQL sqlMode；普通 SQL 生成后增加 dialect sanitizer / validator；dry-run 使用真实 dialect。 |
| P1 | PX01 / ROUTE01 未命中期望 anchored template。 | 业务模板召回不稳定，高价值报表问题可能走普通 SQL。 | 继续优化 template retrieval / businessSignature / positive cues；增加自动化断言。 |
| P1 | OQ guard fallbackReason 未写入 diagnostics。 | 路由实际正确但无法解释，测试脚本失败。 | 在 template guard 命中时稳定落 `fallbackReason` / `reason_codes`。 |
| P1 | 问数完成态 UI 未及时刷新。 | 用户以为任务卡住。 | 修复 polling / streaming 完成事件与 response detail 刷新。 |
| P1 | 数据表 AI 复杂 Filter / Cleaning / Enrichment 受 SQL 方言或字段引用影响失败。 | F-02/F-03/C-01/E-01 均没有保存新版本，但用户无法完成这些 AI 数据处理。 | 统一 spreadsheet rewrite 的 SQL mode；dry-run 前做 dialect normalize；Enrichment 生成字段需绑定真实 source alias。 |
| P2 | Dashboard item preview 对 scope 敏感，完整 KB scope 返回 not found。 | 页面切换继承 URL 参数时可能出现 Dashboard item not found。 | Dashboard API 与页面统一只使用 workspace scope，或后端兼容 item detail runtimeIdentity。 |
| P2 | Diagnostics 默认展示大量 GET_THREADS 空诊断记录。 | 运维排查成本高。 | 默认聚焦 ASK / CHART / SQL Preview，系统调用折叠。 |
| P2 | Chart feedback metadata 没有继承 source response 的 templateDecision。 | 图表反馈治理缺少问数路由上下文。 | chart response feedback 继承 source response traceId/templateDecision。 |
| P2 | 推荐 action 文案/交互不够直观。 | “生成一张图表给我”点击后只是填充输入框，还需再次发送。 | 动作按钮一键执行，或明确标注“填入输入框”。 |
| P3 | Ant Design deprecation warnings。 | 升级维护风险。 | 分批替换 deprecated props。 |
