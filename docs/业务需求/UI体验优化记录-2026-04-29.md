# UI 体验优化记录（2026-04-29 / TiDB 回归）

本记录来自本轮 `docs/业务需求/问数回归测试计划.md` 的 TiDB workspace 复用环境回归。

## 观察范围

- 对话详情页：问数结果、查看数据、导出 CSV/Excel、保存为数据表、反馈按钮。
- 数据看板：固定图表到看板、预览、重命名、删除。
- 数据表：保存表格结果、预览、重命名、删除、AI Filter 操作、History。
- 调用诊断：成功问数、外部依赖阻断、失败问数。

## 体验问题与建议

| 场景 | 现象 | 影响 | 严重度 | 建议 | 状态 |
| --- | --- | --- | --- | --- | --- |
| 数据看板 item 操作 | KB scoped 图表固定后，预览/重命名/删除曾返回 Dashboard item not found / Dashboard not found | 用户会误以为固定失败或无法管理已固定图表 | P1 | dashboard item API 按请求 scope 校验，兼容 KB scoped dashboard | 已在本轮修复并复测通过 |
| 普通问数路由 | OQ06/OQ08 被业务模板强锚定，返回不符合问题意图的数据 | 用户问普通 SQL/明细问题时容易得到“看起来正确但实际不对”的宽表 | P1 | 增强模板 negative cues；普通订单/登录反查类问题优先走普通 SQL | 待修复 |
| 缺参数 LING01 | “统计渠道990011...” 缺少租户时没有先澄清，最终 SQL 解析失败 | 用户不知道缺少什么参数，失败体验不清晰 | P1 | 缺必填参数时先澄清或提示需要租户平台，不进入 SQL 执行 | 待修复 |
| LLM provider 429 | 图表/问数偶发 socket/rate limit，需要重试后通过 | 长任务等待后失败，用户不确定是系统问题还是外部服务波动 | P2 | 前端提示“模型服务繁忙，可重试”，提供一键重新生成；后端做 provider fallback/退避重试 | 待完善 |
| T11 图表类型可观测性 | 图表状态 FINISHED，但 chartType 为空 | 看板固定和回归记录难以判断图表类型 | P2 | chart detail 中补齐 chartType 或提供稳定 recommendedDisplay/type | 待完善 |
