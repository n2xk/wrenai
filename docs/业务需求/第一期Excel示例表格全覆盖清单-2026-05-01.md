# 第一期 Excel 示例表格全覆盖清单（2026-05-01）

## 1. 结论

源文件：`docs/业务需求/第一期数据报表需求V1.xlsx`。

- 工作簿共有 **7 个 sheet**：`表格汇总` + 6 个报表需求 sheet。
- `表格汇总` 是需求索引，不作为问数结果验收表。
- 6 个报表需求 sheet 中共有 **11 个需要问数结果复现的示例表格**。
- 当前 T01~T13 / FC01~FC12 能覆盖主要业务口径，但**还不能声明“核心用例已经完整覆盖并且查出来是和 Excel 一样的表格”**。
  - 主要原因：现有核心用例多数是“语义/指标口径覆盖”，返回结构常为长表；Excel 示例里有宽表、透视表、汇总行、分组表头、D1~D360 周期列、全游戏类型列、全首存金额桶列。
  - 因此需要新增 FT01-FULL~FT11-FULL 作为“Excel 示例表格 1:1 结构覆盖用例”，并新增 FT01-D~FT11-D 作为缺外部数据时优先验证的降级用例。DEGRADED 版必须保存为数据表资产，用来覆盖保存/资产化链路；FULL 版默认只验证问数结果同形，不强制逐张保存数据表。

## 2. “查出来一样的表格”的验收定义

本轮“完全覆盖查询出来一样”按业务验收定义为：

1. **同一业务粒度**：日期 / 区间 / cohort / 用户分层 / 游戏类型 / 首存金额桶等粒度与 Excel 示例一致。
2. **同一字段集合与方向**：Excel 是宽表的，问数结果也必须宽表；Excel 是分层行的，问数结果也必须保留对应分层行。
3. **同一汇总规则**：汇总行必须存在；金额相加、人数按口径去重或相加、比例按汇总后的分子分母重算，不取每日平均。
4. **同一派生公式口径**：ROI、渠道收入、杀率、投充比、UV 下载率、UV 注册率、首存成本、占比、环比等公式必须一致。
5. **同一排序与完整枚举**：周期列、游戏类型列、首存金额桶列必须按 Excel 顺序输出；无数据枚举项按产品约定输出 0 / 空值，但不能丢列。
6. **不要求一致的内容**：不要求 1:1 复刻 Excel 单元格样式、合并单元格、颜色、字体；但页面预览、导出 CSV/Excel、保存到数据表后的列名和行列结构必须一致。

## 4. DEGRADED 版：缺外部数据时优先验证的降级用例

DEGRADED 版用于当前外部数据源尚未补齐时先验证“可查询、可解释、可保存”的主链路。它不是 Excel 同形验收，不能替代 FULL 版通过结论。

### 4.1 降级原则

| 规则 | 要求 |
| --- | --- |
| 不编造外部指标 | 投放金额、访问PV、访问UV、下载点击UV 等缺失指标不能填 0，也不能用内部字段替代 |
| 派生指标同步去掉 | ROI、首存成本、UV下载率、UV注册率等依赖外部指标的列必须去掉，或明确标为“不可计算” |
| 尽量保留内部业务粒度 | 日期、渠道、cohort、TOPN/非TOPN、游戏类型、首存金额桶等内部可查维度尽量保留 |
| 结果明确标记 | 问题、结果说明、保存资产名都必须带 `降级` 或 `内部指标`，避免被误认为 FULL 验收通过 |
| 不计入同形覆盖率 | 降级通过只能计入 DEGRADED 覆盖率；FULL 覆盖率仍以原始 Excel 字段和表形为准 |

### 4.3 推荐验证顺序

1. 先执行 `FT01-D~FT11-D`，确认缺外部数据时至少能产出内部可查询结果并保存数据表。
2. 对 DEGRADED 失败项，优先修“误阻断 / SQL 不可执行 / 保存失败 / 结果串 thread”等主链路问题。
3. 外部数据补齐或外部数据输入链路可用后，再执行 `FT01-FULL~FT11-FULL`。
4. 最终报告必须分开统计：`FULL 同形覆盖率`、`DEGRADED 可用覆盖率`、`已保存 FULL 资产数`、`已保存 DEGRADED 资产数`。

## 5. 数据表保存要求

保存责任按版本拆分：

| 版本 | 是否必须保存数据表 | 说明 |
| --- | --- | --- |
| DEGRADED：FT01-D~FT11-D | 必须保存 | 用于优先验证“缺外部数据时仍可查询、预览、保存”的主链路和数据表资产化体验 |
| FULL：FT01-FULL~FT11-FULL | 默认不强制保存 | 用于验证原始 Excel 示例表格是否能同形查询出来；只需记录 thread / response / SQL / 预览或导出证据 |
| FULL 资产化扩展 | 可选或抽样保存 | 只有当本轮目标还包括“FULL 结果也能沉淀为数据表资产”时，才逐张或抽样保存 FULL 结果 |

验收记录必须区分 DEGRADED 与 FULL，并包含：

| 检查项 | 要求 |
| --- | --- |
| 保存入口 | 只允许表格型结果使用“保存为数据表”；不能固定到数据看板 |
| 资产命名 | DEGRADED 使用 `FTxx-D_...内部指标/降级...` 名称；若额外保存 FULL，则使用 `FTxx_...` 名称，避免和普通回归资产或 DEGRADED 资产混淆 |
| 数据表列表 | 保存后无需刷新或刷新后必须能看到新资产；若需要手动刷新，必须记录为体验问题 |
| 数据表详情 | 默认选中新保存/第一个 item，并展示同一张表 |
| 查看 SQL | SQL 弹窗可打开，SQL 与来源问数一致 |
| 预览数据 | DEGRADED 保存后需校验数据表预览与问数结果一致；FULL 默认校验问数结果预览/导出与 Excel 示例同形 |
| 导出 | CSV / Excel 导出结构与数据表预览一致 |
| 后续操作 | 至少验证重命名；删除可在证据采集完成后执行 |

## 6. 与现有用例体系的关系

- T01~T13：证明核心 SQL 模板/业务口径能跑通。
- FC01~FC12：补齐第一期需求的字段、维度、周期、外部数据、图表场景。
- **FT01-FULL~FT11-FULL：证明 Excel 示例表格本身被完整复现；默认只要求问数结果同形，不强制保存为数据表资产。**
- **FT01-D~FT11-D：证明缺外部数据时可以安全降级为内部指标结果，并保存为数据表资产；不计入 FULL 同形覆盖。**

最终报告只有同时满足下面条件，才能写“第一期数据报表需求V1.xlsx 已完全覆盖”：

1. T 主链路通过。
2. FC 增量场景通过或明确列为产品/数据待开发缺口。
3. FT01-FULL~FT11-FULL 全部执行，且每张问数输出表与 Excel 示例表格结构一致。
4. FT01-D~FT11-D 全部保存为数据表资产并完成数据表详情验收。
5. FULL 结果是否保存为数据表资产作为独立扩展结论统计；未保存 FULL 不影响“查询同形覆盖”结论，但不能声明“FULL 资产化逐张覆盖”。
6. 若本轮只执行 FT01-D~FT11-D，只能声明“降级可用覆盖”，不能声明“原始 Excel 示例表格完全覆盖”。
