# 普通问数补充用例

## 1. 文档目的

本文件用于补充一组 **不以命中业务 SQL 模板（sql_pair / anchored template）为主要目标** 的普通问数用例。

现有 T01~T15 和 FC01~FC12 主要用于验证 `docs/业务需求/第一期数据报表需求V1.xlsx` 的业务报表场景，因此大部分会刻意命中：

- SQL 模板 / sql_pair
- 分析规则 instruction
- 业务词典
- 外部数据依赖

这类用例适合验证“业务知识是否覆盖”。但产品还需要证明：即使用户提出普通探索型问题、明细查询、简单聚合、表结构理解或模糊问题，系统也能走普通语义层 / text-to-SQL / 澄清链路，而不是所有问题都被强行锚定到业务模板。

---

## 2. 普通用例路由验收口径

普通用例不要求全部避开分析规则，因为全局规则可能会自然参与回答边界；但它们 **不应全部命中 anchored sql_pair**。

建议验收规则：

| 项 | 通过标准 |
| --- | --- |
| SQL 模板 | 单个普通问题可以命中模板，但不是通过条件；整组 OQ 用例中至少 60% 不应为 `templateDecision.mode = anchored_template` |
| 分析规则 | 可以命中全局规则或口径规则，但不应依赖某个业务模板才能答对 |
| 输出类型 | 允许普通 SQL 结果、表结构说明、澄清问题、无法回答说明 |
| diagnostics | 记录 `templateDecision`、`sqlSource`、是否使用 sql_pair、是否触发规则 |
| 正确性 | 以本文件的关键对账点为准；不得为了避开模板而牺牲准确性 |
| 失败判定 | 若所有普通用例都被同一类业务模板锚定，说明模板匹配过强，需要调整 negative cues 或普通问数路由 |

## 3. 普通问数用例清单

### OQ01：原始充值订单日聚合

| 项 | 内容 |
| --- | --- |
| 类型 | 普通 SQL 生成 / 单事实表聚合 |
| 推荐问题 | 不用业务报表模板，直接查询租户平台990001下渠道990011在2026-04-01到2026-04-07每天成功充值订单笔数和充值金额 |
| 路由预期 | 优先普通 text-to-SQL；不要求命中 T01 |
| 关键对账点 | 2026-04-01：2 笔 / 30；2026-04-02：3 笔 / 2080；2026-04-03：3 笔 / 238；2026-04-06：2 笔 / 400 |
| 通过标准 | 返回日级充值订单笔数和金额；不把失败充值算入 |

## 4. 与业务覆盖用例的关系

| 用例集 | 主要目的 | 是否期望命中 sql_pair | 是否期望命中分析规则 |
| --- | --- | --- | --- |
| T01~T15 | 第一阶段业务报表核心链路 | 是，T05/T14/T15 为阻断例外 | 是 |
| FC01~FC12 | Excel 全字段/全维度/全周期补充 | 多数会命中或扩展模板 | 是 |
| OQ01~OQ10 | 普通问数与泛化能力 | 不强制；多数应走普通生成/语义层 | 可选，不作为通过条件 |

推荐最终回归报告同时展示三类结果，避免把“模板覆盖率高”误判为“普通问数泛化能力好”。
